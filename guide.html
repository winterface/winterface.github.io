Buttons

  • Primary Call to Action, positive. Used for performing actions like saving, uploading, applying, submitting, etc.

    Primary Button

  • Default Call to Action, used for actions that don't need to stand out as much as the primary call to action. For example, closing a window,

    Default Button

  • Close, Cancel, Delete button. Used for when you have an action that requires caution for the user, negative.

    Cancel, Close, Delete Button

  • On Hold Status Button, used within tables to indicate something is on hold. Width is automatically adjusted based on the column width. In this example I've constrained the button with a col-md-2 only for display purposes.

    On Hold


  • Pending Status Button, used within tables to indicate something is pending. Width is automatically adjusted based on the column width. In this example I've constrained the button with a col-md-2 only for display purposes.

    Pending


  • Closed Status Button, used within tables to indicate something is closed. Width is automatically adjusted based on the column width. In this example I've constrained the button with a col-md-2 only for display purposes.

    Closed


Forms

The form tag uses "form-horizontal" class to format form fields into a 2 column layout

Input, select, and textareas use the class "form-control". This provides all the styling needed for the fields

Labels for the form fields use the class "control-label".

Tables

Tables are wrapped in a div called "table-responsive" this allows the table to have mobile support via touch.

Tables use two classes, "table table-striped". This provides all of the styling for the rows, columns, and hover effects.
